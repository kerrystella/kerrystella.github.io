为什么Java中的String设计成不可变的

发表于 2020-03-13 更新于 2022-08-24 分类于 技术总结

原文出处：https://www.programcreek.com/2013/04/why-string-is-immutable-in-java/

在Java中String类是不可变的(final)。简单来说，一个不可变的类就意味着他的实例是不可修改的，实例的所有信息都是在实例创建的时候被初始化并且不可被修改。不可变类的设计有很多优点。这篇博文主要从内存，同步和数据结构的角度来具体说明这种不可变的概念。

决策准绳之我的价值观

发表于 2020-01-01 更新于 2022-08-23 分类于 人生感悟

价值 - MUST

价值涵盖三个层面：客户价值、组织价值和个人价值。

  • 客户价值

    宏观层面是降本增效，微观层面可以体现为缩短流程、减少人工参与、避免用户误操作。具体分析客户价值的时候需要结合微观和宏观综合考虑

  • 组织价值

    具体体现为带给企业和团队的增益。可以是增加销售额，降低实施成本，也可以是提升团队技能或者增强团队凝聚力

  • 个人价值

    从自身出发考虑行动的收益。可以是收入的增加，可以是技能的提升，也可以是提升幸福感

创新 - BETTER TO HAVE

创新可以从理论、技术和解决方案的角度来衡量。

  • 理论创新

    理论的创新会引发技术和解决方案的创新，从而引领行业的新思潮。例如云计算大行其道的时候，提出边缘计算的思路就是一种理论的创新

  • 技术创新

    通过新技术践行理论创新，可以认为是技术创新。例如KubeEdge和OpenYurt这些开源技术就是紧跟边缘计算思潮的技术创新

  • 解决方案创新

    解决方案创新体现的是一种整合的智慧，通过已有的技术组合创造性的解决生产中的难题或者创造价值。例如利用InfluxDB的订阅机制实现边缘数据库的备份和转发，Canal模拟MySQL的slave进行日志同步都可以认为是解决方案层面的创新

先进 - BETTER TO HAVE

先进性主要体现在技术和思想两个方面。应该努力避免技术定格和思想僵化。

例如Java已经出到16了还抱着8就是一种技术的定格，技术定格的后果是难以引导或者追随创新，最终因为无法创造价值而被淘汰。

思想的僵化也是同样的道理，边缘计算、工业互联网的浪潮中还抱着传统的产品化思想，自我设限的后果就是无路可走。
